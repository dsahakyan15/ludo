# План реализации: Улучшение визуального стиля и анимация

## 1. Базы игроков (Player Bases)
- В `App.tsx` изменим генерацию сетки. Вместо отрисовки 36 отдельных ячеек для каждой базы, мы будем пропускать эти координаты в цикле и создадим 4 больших контейнера `div` с использованием CSS Grid (например, `grid-area: 1 / 1 / 7 / 7` для зеленой базы).
- В `App.css` оформим эти контейнеры в соответствии со скриншотом:
  - Основной цветной фон.
  - Большой белый прямоугольник со скругленными углами внутри.
  - 4 круглых серых «слота» для размещения пешек (абсолютно позиционированные внутри белого квадрата).

## 2. Фигурки в виде пешек (Pawns)
- Текущие `.piece` кружочки будут заменены на SVG-изображения пешек (`Pawn` компонент).
- SVG будет иметь объемный вид за счет тени (`drop-shadow`) и белого блика на «голове» пешки. Цвет пешки будет привязан к команде.

## 3. Абсолютное позиционирование фишек
- Фишки будут вынесены из ячеек сетки. Они будут рендериться как прямые потомки контейнера `.board`.
- Позиция каждой фишки (`top` и `left`) будет вычисляться в пикселях (или процентах/ячейках, например `left: calc(c * var(--cell-size))`) на основе их текущих координат в сетке.
- Это позволит использовать CSS `transition` для плавного перемещения фишек.

## 4. Анимация ходьбы (Walking Animation)
- Функция `movePiece` будет переписана для поддержки асинхронного пошагового выполнения.
- Вместо мгновенного изменения координаты на конечную, позиция фишки (`pos`) будет увеличиваться на 1 за раз в цикле с задержкой (например, 200 мс).
- Во время перемещения фигурке будет добавляться CSS-класс `.moving`. Этот класс будет запускать анимацию `bounce` (подпрыгивание по оси Y) для самой SVG-пешки, создавая реалистичный эффект "шага" с клетки на клетку.
- Игровая логика (проверка на рубку, условия победы, переход хода) будет выполняться только после завершения всего цикла шагов.

## 5. Доработка
- Будет добавлено смещение для фишек, находящихся на одной клетке (чтобы они не перекрывали друг друга полностью), но теперь с учетом их абсолютного позиционирования.

## Верификация:
- Убедиться, что базы выглядят как на предоставленном скриншоте (крупные цветные квадраты с 4 слотами).
- Убедиться, что фишки выглядят как пешки.
- Убедиться, что при ходе фишка делает последовательные шаги по клеткам, плавно перемещаясь и подпрыгивая.
